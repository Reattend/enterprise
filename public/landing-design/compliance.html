02 / Controls

Six layers, zero "trust me bro."

Every claim on this page maps to a control in our SOC 2 report. The full mapping is in the trust report; the headlines are below.

01 / Identity

Identity & access

SSO is mandatory. Roles map to org structure. Service accounts rotate keys on a 30‑day timer; nothing recalls without a name attached.

  • SAML 2.0 · OIDC · SCIM 2.0
  • RBAC + per‑memory ACLs
  • Session bound to device + IP range
  • Hardware key (WebAuthn) optional
02 / Encryption

Encryption

AES‑256 at rest, TLS 1.3 in transit, envelope keys rotated quarterly. BYOK on Enterprise; we never see your data‑encryption key.

  • AES‑256‑GCM · TLS 1.3 only
  • HSM‑backed root keys (FIPS 140‑3)
  • Per‑tenant data keys · quarterly rotation
  • BYOK via AWS KMS / GCP KMS
03 / Audit

Audit & provenance

Every recall returns a citation chain. Every admin action goes to your SIEM in under 60 seconds. Logs are tamper‑evident with a Merkle digest published daily.

  • Citation graph on every answer
  • Splunk · Datadog · Snowflake export
  • 7‑year retention default
  • Tamper‑evident hash chain
04 / Privacy

Privacy & retention

Your prompts and documents do not train any model — ours, or anyone else's. Retention is configurable per memory class; deletion is real, not soft.

  • No training on customer data, ever
  • Per‑memory TTL (30d → indefinite)
  • Right‑to‑erase < 72h SLA
  • PII redaction on ingest (toggle)
05 / Resilience

Resilience

Multi‑AZ by default; multi‑region available. RTO is one hour; RPO is fifteen minutes. We test failover quarterly with full chaos drills.

  • 99.95% uptime SLA · status.reattend.com
  • Quarterly DR test · 4h restore
  • Daily encrypted backups · 35d retain
  • Active/active in two AZs
06 / Vulnerability

Vulnerability mgmt

SAST and SCA on every commit. External pen test twice a year by NCC Group. Public disclosure policy and bug bounty live.

  • SAST · SCA · DAST in CI
  • Pen test · NCC Group · 2x/year
  • Bug bounty · HackerOne
  • Patch SLA · critical < 48h
03 / Data flow

From source to recall — encrypted at every step.

Here's exactly what happens when a document enters Reattend and a teammate later asks a question against it. Nothing is stored in the clear. Nothing leaves the boundary you choose.

STEP 01

Ingest

Document captured from source (Slack, Drive, repo). TLS 1.3 over the wire. Source ACL preserved 1:1.

TLS 1.3 · ECDHE‑X25519
STEP 02

Embed & index

Vectors generated in your region. PII redaction applied on flagged classes. Object‑store write is encrypted.

AES 256‑GCM · per‑tenant DEK
STEP 03

Recall

Query checked against ACLs before retrieval. Only chunks the asker can see are decrypted into RAM. Nothing else.

ACL check · zero leakage
STEP 04

Audit

Citation chain hashed and chained into the daily Merkle root. Event streams to your SIEM. Plaintext is dropped.

SIEM < 60s · Merkle daily

07 / Questions

The questions your security team will ask.

Twelve answers we've given a thousand times. If yours isn't here, write to trust@reattend.com.

Do you train any model on our data?+

No. Customer prompts, documents, embeddings, and recall traffic are excluded from all training — ours and our LLM subprocessors'. Anthropic and OpenAI both run under zero‑data‑retention agreements. The only exception is opt‑in feedback your admins explicitly route to us in writing.

Where is data stored, exactly?+

Whichever region you pick at sign‑up — and only that region. Embeddings, object storage, audit logs, and backups all stay inside the boundary. Cross‑region replication is opt‑in and only between regions you've explicitly approved.

Can we bring our own keys?+

Yes, on Enterprise. We support BYOK via AWS KMS and GCP KMS. Reattend never holds the data‑encryption key — we hold a wrapping key you can revoke at any time, which makes the tenant cryptographically inaccessible within seconds.

What's the incident notification SLA?+

Under four hours for any P1 confirmed incident affecting your tenancy, with a written follow‑up inside 72 hours. Status incidents (degraded service, not security) post to status.reattend.com in real time.

How do you handle a right‑to‑erase request?+

Admins can issue a full‑subject delete from the dashboard; we propagate to vector index, object store, audit log archive, and all backups within 72 hours. We return a signed deletion receipt with a Merkle proof you can hand a regulator.

Do you support air‑gapped / on‑prem?+

Yes, on Enterprise. The full stack — ingest, vector index, recall — ships as a Helm chart that runs in your VPC or on‑prem cluster. The only outbound call is a license heartbeat, which can be replaced with an offline license key for fully air‑gapped deployments.

How is access logged?+

Every recall, edit, share, and admin action emits an event. Events stream to your SIEM (Splunk, Datadog, Snowflake) inside 60 seconds and are retained 7 years by default. The event chain is hashed daily into a Merkle root we publish — so a logged event can't be silently rewritten.

Who has access on your side?+

Production access is gated by hardware key, per‑request approval, and a 30‑minute time‑box. All sessions are recorded. The customer support team has zero access to memory contents — they see metadata and audit, never the document body.

How is the LLM prevented from leaking documents?+

Recall checks ACLs before retrieval. The model only ever sees chunks the asker is authorized to read. The model layer is stateless across queries; nothing persists in the inference path. Prompts and outputs are zero‑retained on the LLM side.

What if a subprocessor changes?+

We give 30 days' written notice via the trust@reattend.com mailing list before any new subprocessor is added or an existing one's scope expands. You can object during that window; we offer reasonable mitigations or, on Enterprise, a region carve‑out.

Pen test cadence?+

Twice yearly with NCC Group covering web, API, infrastructure, and a focused review of the recall path. Findings are remediated by severity SLA (critical < 48h). Sanitized summaries go in the trust report; the full report is available under NDA.

How do we run a security review?+

Email trust@reattend.com with your assessor and we'll return the trust packet (SOC 2, ISO, pen test, DPA, BAA, whitepaper) inside 24 hours. We respond to standardized questionnaires (CAIQ, SIG Lite, VSAQ) without a custom rewrite — answers are pre‑mapped to controls.
